# Agent Triage Labels

> 本文件约定本地 Markdown 工作项的状态标签，避免每轮对“能不能做”“谁来确认”“是否已经完成”重新发明一套说法。

## 状态标签

| 标签 | 含义 |
| --- | --- |
| `needs-triage` | 还没判断优先级、范围、风险或基线。 |
| `needs-info` | 缺少用户确认、真实环境信息、账号权限或外部状态。 |
| `ready-for-agent` | 范围清楚，agent 可以直接实现或验证。 |
| `ready-for-human` | 需要用户手动验收、视觉确认、真实账号操作或业务判断。 |
| `blocked` | 当前无法继续推进，且阻塞原因已明确记录。 |
| `done` | 本地修改和要求的验证已完成；若涉及生产，还必须单独记录 Docker/内网验证结果。 |
| `wontfix` | 明确不做，通常因为不符合当前基线、被回滚、风险过高或用户取消。 |

## 使用规则

- 标签只描述当前工作项，不自动代表代码已提交、已推送或生产已同步。
- 涉及 `http://192.168.0.39:3456/` 的行为，必须额外记录 Docker 是否重建、容器是否 healthy、线上路径是否命中新版本。
- 从历史日志恢复任务时，先检查该任务是否仍存在于当前 Git 基线；不在当前基线里的任务先标记为 `needs-triage`。
