# 电商套图 Agent Skill Markdown 生成规范

## 用途

本规范用于后台「电商套图 Agent」的设计师 skill Markdown。Markdown 只作为提示词上下文传给文本模型，不执行代码，不作为前端页面渲染。

## 后台字段

- `id`：稳定英文小写，建议 kebab-case 或短 id，例如 `gloria`。
- `name`：前台显示名称，例如 `Gloria`。
- `description`：一句话描述设计师能力，会出现在前台 skill 下拉中。
- `enabled`：是否启用。
- `markdown`：本文档格式的完整 skill 内容。

## 推荐 Markdown 结构

每个 skill 文档建议保持以下结构，便于模型稳定读取。

```markdown
# Skill 名称

## 角色定位
说明设计师身份、核心优势、适用品类和主审美方向。

## 设计目标
说明该 skill 要优先优化什么，例如转化率、品牌质感、信息清晰度、生活方式氛围。

## 动态板块策略
要求模型根据产品图、参考图和用户需求动态生成 3-5 个板块，不套固定模板。

## 图像理解规则
说明产品图和参考图分别如何使用。

## 提示词写法
说明每个板块 prompt 应包含哪些视觉要素、构图、光影、材质、文案约束。

## 负面约束
列出禁止内容，例如虚构功效、虚构认证、价格、二维码、水印、乱码文字、替换产品等。

## 输出检查清单
让模型在返回前检查商品识别、参考迁移、板块差异、合规和可执行性。
```

## 动态板块规则

- 板块由模型根据本次输入动态生成，不使用固定「首屏主视觉 / 核心卖点图 / 效果展示图 / 科技解析图 / 使用场景图」五件套。
- 默认生成 3-5 个板块。
- 每个板块必须有明确商业目的，不能只是换一个背景。
- `sectionKey` 用英文 kebab-case，例如 `fresh-fragrance-kv`。
- `sectionName` 用中文短名称，例如 `留香主视觉`。
- `title` 可以比 `sectionName` 更具体，例如 `72小时留香转化主图`。

## 产品图与参考图优先级

- 产品图优先级最高，用于锁定真实主体、包装结构、颜色、材质、Logo、产品名和可辨识文字。
- 参考图只迁移构图、光影、氛围、版式节奏、背景方向和卖点表达方法。
- 禁止把参考图中的其他品牌、产品、Logo、包装文字或不存在的卖点迁移到用户产品上。

## 合规边界

- 不虚构价格、活动、平台认证、检测认证、医学功效、国家背书、销量数据、二维码、水印。
- 不生成不可控长文案；需要文字时用短标签、短标题、清晰中文。
- 不夸大效果，不把普通产品写成医疗、药品或绝对功效。
- 不要求模型输出 HTML、脚本、外链或任何需要执行的内容。

## 好 skill 的标准

- 角色差异明显：每个设计师有不同审美和决策偏好。
- 可执行：能指导模型生成具体板块，而不是空泛夸奖。
- 保守合规：不为了效果虚构用户没有提供的信息。
- 服务当前产品：板块根据图片和需求变化，而不是固定模板排列。
