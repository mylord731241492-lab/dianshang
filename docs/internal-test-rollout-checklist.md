# 内网测试到服务器部署清单

## 当前策略

先完成公司内网测试，确认功能、数据、配置和管理后台稳定后，再部署到服务器。服务器部署不是当前第一步，避免在前端/后台/接口还在验收时过早引入域名、HTTPS、生产密钥和运维复杂度。

## 阶段 1：内网测试

| 检查项 | 状态 | 验收方式 | 说明 |
| --- | --- | --- | --- |
| 一体服务启动 | 进行中 | `npm start` 或 Docker Compose | 无真实 key 也必须 mock 跑通。 |
| 健康检查 | 进行中 | `/api/health` | 应显示数据库、provider、mock/real 状态。 |
| 前端入口 | 进行中 | `/`、`/template-image`、图库、用户中心、后台 | 不空白、入口可点击、有合理反馈。 |
| 后台配置持久化 | 进行中 | 设置、API 线路、模型价格、模板工作流写入后重启 | 当前通过 SQLite `app_state` 保存。 |
| 模板/图库闭环 | 进行中 | mock 生成后图库历史可见 | 不要求真实 AI。 |
| 画布 | 待复核 | 本地 JSON 导入/导出、本地文件夹授权 | 当前明确本地优先，不作为服务端部署阻塞项。 |
| New-API | 待复核 | 配置真实 New-API 后测试连接 | 内网测试阶段可先保持 mock。 |
| Docker 持久化 | 待复核 | 容器重启后 `/api/health` 和后台配置仍可读 | 需要有 Docker 的机器执行。 |

## 阶段 2：服务器部署

| 检查项 | 状态 | 验收方式 | 说明 |
| --- | --- | --- | --- |
| 生产 `.env` | 待实现 | `REQUIRE_ENV_FILE=true`、`REQUIRE_PRODUCTION_SECRETS=true` | `JWT_SECRET` 和 New-API token 不允许是占位值。 |
| Nginx/HTTPS | 待实现 | 域名访问、反代 headers、上传大小限制 | 内网稳定后再配置。 |
| 备份恢复 | 待实现 | SQLite、uploads、logs 备份和恢复演练 | 先停服务或保留 WAL 文件。 |
| 访问权限 | 待实现 | 管理后台账号、服务器防火墙、内网/公网策略 | 按公司实际网络决定。 |
| New-API/CPA | 待实现 | New-API token、模型白名单、CPA 后置渠道 | 本项目不内置 CPA，不重复造轮子。 |
| 监控日志 | 待实现 | 健康检查、容器日志、磁盘容量 | 先轻量，规模化再接集中监控。 |

## 不在当前阶段强做

- 不把画布本地 JSON 改成强服务端存储。
- 不引入 Postgres、Redis、BullMQ、独立 Worker。
- 不在本项目里重造 New-API/CPA 的网关、额度、账号池和统计能力。
- 不默认开启真实 AI、支付、邮件或云存储。
