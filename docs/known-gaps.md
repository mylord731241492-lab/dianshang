# 已知缺口与后续路线

## 当前仍为 mock 的能力

- 真实 AI 生图、反推提示词、视频生成。
- 支付订单、充值、发票和回调。
- 真实邮件发送。
- 真实云存储和跨设备素材同步。
- API 线路新增/编辑/删除的持久化。
- 后台模板工作流保存到文件或数据库的长期持久化。
- 画布高级 AI 工具：扩图、消除、局部修改、参考图融合的真实图像编辑效果。

## 前端 1:1 尚需精修

- 逐页与线上站点做桌面/移动端截图 diff。
- 首页滚动行为、图库锚点、历史弹窗动效仍需逐项比对。
- 画布内部复杂交互需要长流程验收：新增节点、连线、上传、生成、保存、恢复。
- 移动端后台和画布布局需要单独验收。

## 后端真实服务接入顺序

1. AI provider：先接文本反推，再接图片生成，最后接视频生成。
2. 文件存储：先本地稳定保存，再接对象存储。
3. 邮件服务：验证码真实发送。
4. 支付：订单创建、支付回调、余额入账、退款。
5. API 线路管理持久化：把线路、模型、价格迁移到 SQLite 表。

## 不建议现在做的事

- 在没有前端源码的情况下大规模重写打包 JS。
- 在视觉未验收前投入真实支付和真实外部模型。
- 默认开启真实服务调用。
- 删除已有 mock 兼容字段。
