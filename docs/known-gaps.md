# 已知缺口与后续路线

## 当前仍为 mock 的能力

- 真实 AI 生图通道已接入 Provider Adapter，但仍未开启 `ENABLE_REAL_AI=true` 做真实联通测试；反推提示词、视频生成仍是 mock。
- 支付订单、充值、发票和回调。
- 真实邮件发送。
- 真实云存储和跨设备素材同步。
- API 线路、模型价格、模板工作流和系统设置目前通过 SQLite `app_state` 持久化；还未迁移为独立业务表。
- 画布高级 AI 工具：扩图、消除、局部修改、参考图融合的真实图像编辑效果。
- New-API 真实联通测试、真实 `/images/generations` 返回结构确认和 CPA 后置渠道配置。
- Docker 容器已可启动，重启后 SQLite 基础持久化已验证；仍需人工浏览器完整点击测试。
- CodeGraph 未在当前项目目录初始化，结构索引工具暂不可用。
- 服务器正式部署前的 `.env` 生产密钥、Nginx/HTTPS、访问权限和备份恢复演练尚未完成；当前先以内网测试为目标。

## 前端 1:1 尚需精修

- 逐页与线上站点做桌面/移动端截图 diff。
- 首页滚动行为、图库锚点、历史弹窗动效仍需逐项比对。
- 画布内部复杂交互需要长流程验收：新增节点、连线、上传、生成；JSON 文件导入已由 `scripts/smoke-canvas-json-ui.ps1` 验证，导出/本地文件夹授权仍需人工确认。
- 画布当前明确保持本地优先，动态项目刷新和本地恢复依赖浏览器本地状态；当前阶段不作为服务端部署阻塞项。
- 画布保存面板默认本地保存时需要用户授权本地文件夹；未授权时提示 `请先选择本地保存文件夹` 属于预期，后续由人工按实际 JSON 导出和本地文件夹授权习惯验收。
- 移动端后台和画布布局需要单独验收。
- 后台创建类弹窗保存成功后不会自动关闭，关闭/取消按钮体验需要前端 polish。

## 后端真实服务接入顺序

1. AI provider：先接文本反推，再接图片生成，最后接视频生成。
2. 文件存储：先本地稳定保存，再接对象存储。
3. 邮件服务：验证码真实发送。
4. 支付：订单创建、支付回调、余额入账、退款。
5. 后台配置独立表：把 API 线路、模型、价格、模板工作流从 `app_state` 迁移到独立 SQLite 表。

## 不建议现在做的事

- 在没有前端源码的情况下大规模重写打包 JS。
- 在视觉未验收前投入真实支付和真实外部模型。
- 默认开启真实服务调用。
- 删除已有 mock 兼容字段。
- 7 月前内网版本强行引入 Postgres、Redis、BullMQ、Worker 全套重架构。
- 在 New-API/CPA 已能覆盖网关、额度、统计和账号池时，在本项目里重复实现这些基础设施。
