# 步 0 · 元信息与坐标系

> 这是整套描述的**空间语言**。后面所有"方位""深度""光向""色彩分布"都引用本步建立的坐标系。必须最先做，且贯穿始终。坐标系不立，后面的精度就是空话。

建立四样东西：画幅 → 坐标约定 → 空间锚点 → 景别。

---

## 1. 画幅与朝向

- **宽高比**：如 3:2、16:9、1:1、4:5、9:16（竖）。给出近似比例。
- **朝向**：横构图 / 竖构图 / 方构图。
- 比例本身是信息：9:16 通常是手机/竖屏内容，4:5 常见于社媒信息流，2.39:1 是宽银幕电影。但只记录、不在本层推断用途（那是功能层）。

---

## 2. 坐标约定（两套并用）

精度靠数字，可读靠命名。两套一起用：

**A. 归一化坐标（给机器精度）**
- 横轴 `x`：0（最左）→ 1（最右）
- 纵轴 `y`：0（最上）→ 1（最下）
- 一个点写成 `(x≈0.66, y≈0.40)`。元素跨区域时给范围 `x≈0.5–0.8`。

**B. 三分网格命名（给人读）**
把画面切成 3×3 九宫格，命名：左上 / 上中 / 右上 / 左中 / 正中 / 右中 / 左下 / 下中 / 右下。四个三分交点分别叫"左上交点、右上交点、左下交点、右下交点"。

两者并写，例：`主体在右下交点附近 (x≈0.66, y≈0.66)`。

---

## 3. 空间锚点（决定纵深怎么读）

这几样确立后，深度判断才有参照。逐项取证、给坐标：

- **视平线 / 地平线（horizon / eye level）**：记其在画面中的高度 `y` 值。地平线压得低（y 大）→ 仰拍感、天空多；压得高（y 小）→ 俯瞰感、地面多。若画面无明确地平线（特写、纯图形），写"无明确地平线"。
- **灭点（vanishing point）**：若有线性透视，标灭点位置 `(x, y)`，并说明是一点透视 / 两点透视 / 三点透视。依据＝哪些平行线在向该点收敛。
- **机位高度（camera height）**：虫视（极低机位，向上看）/ 低机位 / 平视（视平线≈主体眼高）/ 高机位 / 俯视 / 顶视（正上方）。依据＝地平线位置 + 物体顶/底面的可见性（能看到桌面顶部＝机位高于桌面）。
- **机位角度（camera tilt/yaw）**：正面 / 侧面 / 斜侧（3/4）/ 背面；有无倾斜（荷兰角，画面歪斜）。

> **取证示范**：`机位判为略高于主体的俯视，依据：能看到桌面台面 (y≈0.55–0.8) 的顶部、地平线被推到 y≈0.25、人物头顶可见而下颌底不可见。`

---

## 4. 景别（shot scale）——并提醒一个命名陷阱

**景别 = 画面取了多大范围 / 主体占多满**，是与"深度分层"不同的另一回事：
- 远景（人在环境中很小）/ 全景（人物全身入画）/ 中景（约膝/腰以上）/ 近景（胸以上）/ 特写（头肩或局部）/ 大特写（眼、唇等局部）。

> **命名陷阱（务必避坑）**：中文里"中景"有两个意思——
> - 作为**景别**：指"取了腰以上的范围"；
> - 作为**深度层**：指"前景和背景之间的中间纵深层"。
>
> 二者完全不同。**本 skill 约定**：深度层一律用「前景 / 中景层 / 背景」（必要时写"中间景"消歧），"景别"词汇专门留给"画面取了多大范围"。写描述时不要让这两个"中景"打架。

景别取证：依据＝主体占画面的比例 + 可见到主体的哪个部位边界。

---

## 本步输出片段（示例格式）

```
画幅 3:2 横构图。
坐标系：x 左0→右1，y 上0→下1。
视平线 y≈0.45（依据：地平线两侧平行线收敛于此高度）。
灭点：一点透视，灭点 (x≈0.58, y≈0.45)。
机位：平视，正面略偏 3/4 侧；无荷兰角。
景别：中景（主体约腰以上入画，占画面高度约 60%）。
```
